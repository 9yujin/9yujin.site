# 7. 마무리

_2022.02.22_

프로젝트가 벌써 마무리 단계에 있다. 21학번 메인 기수 친구들이 만들고 있는 학회 홍보물에 같이 껴서 뿌려질 계획이다. 그 친구들과 시간을 맞춰서 개발을 진행해왔다. 세세한 문구들을 마지막으로 다듬었고 테스트용으로 넣어놓은 것들, 콘솔.로그들, 등등을 지우고 마무리할 준비를 끝냈다. 두둥.

## 메인 랜딩 페이지

![](https://img1.daumcdn.net/thumb/R1280x0/?scode=mtistory2&fname=https%3A%2F%2Fblog.kakaocdn.net%2Fdn%2F2C7gD%2Fbtrt3gtg279%2FK4Ab5UbuLjGKBiAeH6AZZ1%2Fimg.png)

​메인 랜딩 페이지 디자인은 좀 급하게 끝났다. 세로로 긴 형태의 카드뉴스(?)는 많이 만들어봤지만 데스크톱 원페이지 웹사이트는 처음이라.. 생각보다 생각해야할게 많더라. 스크롤 애니메이션은 주안이형이 맡아서 이쁘게 잘 해주었다. 큰 이미지는 [여기서](https://www.figma.com/file/J6HVLxWGuCFgAQUCdWBUsT/%EA%B3%A0%EC%8A%A4%EB%9D%BD-%ED%8B%B0%EC%BC%93%EC%98%88%EB%A7%A4?node-id=1434%3A13934) 볼 수 있다. (피그마)

구현을 하면서 기존 디자인과 달라진 부분들이 많다. 피그마에서 한 디자인과 실제로 한 것과 뭔지 모르게 다른게 있더라. 계속 더 이쁘고 자연스럽게 보이도록 수정하면서 했다. 무엇보다 반응형이 자연스럽게 되도록 노력을 많이 했다. 화면 너비에 따라 보여지는 글의 줄바꿈이 달라졌으면 했다. [react-responsive](https://www.npmjs.com/package/react-responsive) 라이브러리를 이용하면 리액트 컴포넌트 안에서 미디어쿼리에 따라 다르게 렌더링을 할 수 있다.  
​

![](https://blog.kakaocdn.net/dn/caW1uB/btrt3fVs5CJ/Nk1ZDgwLYb4hSSQRpZtxpk/img.gif)

자잘한 완성도를 높이는데 주력했다. 덕분에 나름 괜찮은 페이지가 나온 것 같다.

헤더의 'gosrock'을 클릭하면 스크롤 최상단으로 이동하도록 했었다. 보통은 오른쪽 하단에 두긴 하지만 이런 버튼을 'back to top' 또는 'scroll to top' 버튼이라고 부른다. 우리 동아리 구호의 '정상을 향하여!'를 영어로 '**Toward the Top**' 라고 하는데, 말이 비슷하다는 점에서 한번 재미를 넣어보았다. 뜻이 통하는 말장난. 아주 센스있는 아이디어였다고 자찬을 해본다.

```css
html {
  scroll-behavior: smooth;
}
```

`scroll-behavior: smooth` 속성은 같은 페이지 내에서 이동할 때 스크롤의 움직임을 부드럽게 해준다.  
​

## OG태그, 파비콘

예매 링크를 뿌릴 때 보여질 부분도 신경 써야 했다. og 태그와 파비콘을 바꾸어 주었다.

```jsx
    <meta property="og:title" content="고스락 티켓" />
    <meta
      property="og:description"
      content="3월 10일 오후 7시, 합정 라디오 가가 라이브홀에서 만나요!"
    />
    <meta property="og:image" content="%PUBLIC_URL%/og.png" />
    <link rel="apple-touch-icon" href="%PUBLIC_URL%/logo.png" />
```

public 디렉토리에 있는 파일을 `index.html` 에서 위와 같은 방식으로 접근할 수 있다. `apple-touch-icon` 로 아이폰에서 웹사이트를 '홈 화면에 추가' 했을 때 보이는 아이콘을 설정할 수 있다.
