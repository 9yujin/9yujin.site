## 무엇을 기록할까요?

‘기술 블로그’ 에 글을 쓴다고 생각하고 자리에 앉으면 쉽사리 글이 잘 안써집니다. 거창한 무언가를 적어야 한다는 느낌이 듭니다. 개발하며 참고했던 여러 블로그들처럼 어려운 내용을 멋있게 정리해놓아야만 할 것 같죠.

사실 저는 이미 티스토리에서 블로그를 하고 있어요. [안티프리즈!](https://9yujin.tistory.com) 라는 이름을 달고 있습니다. '이 뜨거운 열정이 식지 않게 꾸준히 공부하자!!' 라는 거창한 뜻은 아니고, 그냥 좋아하는 노래 제목입니다. 기술 블로그에 적는 글도 거창한 내용을 담아야 할 필요는 없습니다. 아무 의미 없는 제 블로그 이름처럼요.

처음엔 개발하며 찾아봤던 지식들을 단순히 정리해보며 시작했어요. context api 같은 라이브러리 사용법을 적기도 했고, https 인증 설정했던 경험을 기록하기도 했습니다. 요즘은 프로젝트를 하면서 고민했던 내용, 삽질했던 내용, 회의록을 적기도 합니다. 1년짜리 회고도 열심히 적었었네요.

조금 더 정리해보겠습니다. 이렇게 매우 다양한 내용이 들어갈 수 있어요. 그냥 하고싶은대로 하면 돼요.

- **오늘 새로 배운 내용** : 매주 진행하는 스터디의 주제나, 과제를 하며 새로 알게 된 부분을 기록해요. 아주 간단한 자바스크립트 문법부터, 새로 써본 라이브러리, 뭐 이런것들을 다 적을 수 있을 것 같아요.
- **고민한 내용** : 개발이나 과제를 하다보면 고민하는 일이 많죠. 무언가를 구현해야 하는데 막혔던 부분, 어찌어찌해서 해결한 방법들, 그 방법까지 도달한 과정 등등을 적을 수 있을 것 같습니다. 해결하지 못했다 하더라도 상관 없어요!!
- **참고한 문서** : 이게 내가 짠건지 구글이 짠건지 헷갈릴 때가 있습니다. 개발하면서 참고했던 글들의 링크를 많이 달아놓아요.
- **회고** : 프로젝트나 스터디, 세미나 등을 참여하며 들은 내용과 소감 등을 적어요.
- 이 외에도 그냥 **다음주 목표**, **개발 얘기**, **코딩 개그**..., **요즘 드는 고민**, **생각** 등등, 아무거나 다 좋아요!!

## 어디에 기록할까요?

**Velog** 또는 **Tistory** 를 사용할 거에요. 큰 차이는 없습니다. 두 플랫폼 모두 개발 블로그로 제일 많이 사용되는 블로그이고 **마크다운** 문법을 사용할 수 있습니다.

어디서 주워들은 말인데, velog는 아이폰같고 tistory는 안드로이드 같다고 합니다. velog는 이쁘고 별다른 세팅 없이 사용할 수 있는 대신 별다른 기능이 없고 커스텀이 불가합니다. 하지만 tistory는 초기 세팅이 어느정도 필요합니다. 대신 본인의 블로그를 마음대로 커스텀할 수 있고 플러그인을 통해 다른 여러 기능을 넣을 수 도 있어요.

자신에게 더욱 잘 맞는 플랫폼을 선택해 블로그를 시작해봅시다!! 과제 제출을 할 때는, 작성하신 글을 md파일로 만들어 커밋해주세요.

## 마크다운

마크다운은 마크업 언어의 일종입니다. 마크업은 글을 좀 더 편하게 쓸 수 있도록 도와주는 도구에요. 송하윤 교수님 수업에서 latex을 써보신적이 있나요?? 그것도 마크업이에요.... 레이텍은 아닐지 몰라도 마크다운은 엄청엄청 편하답니다.

![](https://img1.daumcdn.net/thumb/R1280x0/?scode=mtistory2&fname=https%3A%2F%2Fblog.kakaocdn.net%2Fdn%2FdaZMTj%2FbtrNeF8hXbd%2FMAfxtq83IT6WPIbCRFxXvk%2Fimg.png)

지금 보고 계신 이 글도 마크다운으로 작성 되고 있습니다. `#`, `**`, `-` 등의 태그를 이용해서 제목, 리스트, 인용, 코드 등을 쉽게 작성할 수 있어요.

[참고자료](https://velog.io/@yuuuye/velog-%EB%A7%88%ED%81%AC%EB%8B%A4%EC%9A%B4MarkDown-%EC%9E%91%EC%84%B1%EB%B2%95)를 참고해서 작성해보세요!! 링크에 없는 문법도 찾아서 써보아도 좋아요. 마크다운이 어렵게 느껴지신다면, 노션을 활용해보세요. 노션도 마크다운을 사용하고 있어요. `/`을 입력하면 나오는 여러 메뉴들 대부분은 마크다운을 쉽게 입력할 수 있도록 해줍니다. 물론 노션에만 있는 기능(콜아웃, 페이지 등)은 기존 마크다운에서 지원하지 않아요.

저희는 코드를 쓸 일이 많을테니 추가로 덧붙이자면,

````
```
...code
```
````

백틱(아마 왼쪽 위에 있을거에요) 3개는 코드블럭을,

```
`code`
```

백틱 한개는 `인라인 코드`를 표현할 수 있어요.
